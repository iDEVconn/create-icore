# Plan 6.1: Ant Design Client Template

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add `apps/templates/client-antd` as a sibling to `client-shadcn`. Same `libs/template-shared` foundation, same route tree, same `setNotifier` hook — only the visual layer differs. After this plan: `npm init @idevconn/icore my-app -- --ui=antd` no longer falls back to shadcn; it scaffolds a real Ant Design 6 + dayjs + i18next SPA modelled after `ui-main/apps/client`.

**Architecture:** No new shared logic — `template-shared` already exposes everything the antd template needs (auth store, i18n bootstrap, AbilityProvider, useNotify, useDraft, LandingPage). The antd template wires its own visual layer:

- **Routes** — identical file tree to `client-shadcn` (`/`, `/login`, `/_dashboard`, `/_dashboard/dashboard`, `/_dashboard/profile`).
- **Layouts** — `Layout` / `Layout.Header` / `Layout.Sider` / `Layout.Content` / `Layout.Footer` from antd, with `Affix` on the header (matches `ui-main/apps/client/src/layouts/mainLayout.tsx`).
- **PageLayout** — `Descriptions` header + `Spin` loading overlay + `<Can>` gate, equivalent to `ui-main/apps/client/src/layouts/page.layout.tsx`.
- **LoginForm / ProfileForm** — `Form` + `Input` + `Button` + `Form.Item` validation.
- **AccessDeniedPage** — `Result` component with status `403`.
- **Notify host** — `notification.useNotification()` registered into `setNotifier` so app code stays library-agnostic.

**Tech Stack:** antd ^6.0, `@ant-design/icons` ^6, dayjs (antd peer), `classnames`, the same TanStack Router/Query, Zustand, i18next, CASL, useDraft pieces from `template-shared`.

**Source spec:** `docs/superpowers/specs/2026-05-28-icore-design.md` § Client UI library — bootstrap-time choice.

import { defineConfig } from 'vite';
import react from '@vitejs/plugin-react';
import { tanstackRouter } from '@tanstack/router-plugin/vite';
import fs from 'node:fs';
import { fileURLToPath } from 'node:url';
import path from 'node:path';

const here = path.dirname(fileURLToPath(import.meta.url));
const rootPkgPath = new URL('../../../package.json', import.meta.url);
const rootPkg = JSON.parse(fs.readFileSync(rootPkgPath, 'utf-8')) as {
  version: string;
  dependencies?: Record<string, string>;
  devDependencies?: Record<string, string>;

const ver = (name: string): string =>
  rootPkg.dependencies?.[name] ?? rootPkg.devDependencies?.[name] ?? '?';

export default defineConfig({
  root: here,
  cacheDir: '../../../node_modules/.vite/client-antd',
  define: {
    'import.meta.env.VITE_APP_VERSION': JSON.stringify(rootPkg.version),
    'import.meta.env.VITE_DEP_REACT': JSON.stringify(ver('react')),
    'import.meta.env.VITE_DEP_ANTD': JSON.stringify(ver('antd')),
    'import.meta.env.VITE_DEP_VITE': JSON.stringify(ver('vite')),
    'import.meta.env.VITE_DEP_TANSTACK_ROUTER': JSON.stringify(ver('@tanstack/react-router')),
    'import.meta.env.VITE_DEP_TANSTACK_QUERY': JSON.stringify(ver('@tanstack/react-query')),
    'import.meta.env.VITE_DEP_ZUSTAND': JSON.stringify(ver('zustand')),
    'import.meta.env.VITE_DEP_CASL': JSON.stringify(ver('@casl/ability')),
  },
  plugins: [
    {
      name: 'inject-app-version-meta',
      transformIndexHtml(html) {
        return html.replace('%APP_VERSION%', rootPkg.version);
      },
    },
    tanstackRouter({
      target: 'react',
      autoCodeSplitting: true,
      routeFileIgnorePattern: '(__tests__|\\.test\\.(t|j)sx?$)',
    }),
    react(),
  ],
  server: { port: 4201 },
  preview: { port: 4201 },
  build: { outDir: 'dist', emptyOutDir: true, reportCompressedSize: true },
});

## Task 3: Bootstrap (`main.tsx`) + landing route

- [ ] **Step 1: Notify wiring (antd `notification.useNotification`)**

Create `apps/templates/client-antd/src/lib/notify.ts`:

```ts
import { notification, type NotificationInstance } from 'antd';
import { setNotifier, type NotifyOptions } from '@icore/template-shared';

let api: NotificationInstance | null = null;

export function bindNotifier(instance: NotificationInstance): void {
  api = instance;
  setNotifier({
    success: (title, opts) =>
      api?.success({ message: title, description: opts?.description, duration: opts?.duration }),
    error: (title, opts) =>
      api?.error({ message: title, description: opts?.description, duration: opts?.duration }),
    info: (title, opts) =>
      api?.info({ message: title, description: opts?.description, duration: opts?.duration }),
    warning: (title, opts) =>
      api?.warning({ message: title, description: opts?.description, duration: opts?.duration }),
  });
}
```

(Unlike sonner — which has a singleton `toast` — antd requires the host element be mounted inside React. Bind happens in MainLayout via `notification.useNotification()`.)

- [ ] **Step 2: main.tsx**

Replace `apps/templates/client-antd/src/main.tsx` with the equivalent of the shadcn bootstrap (commit `e29a4f7`):

```tsx
import './globals.less';
import { StrictMode } from 'react';
import { createRoot } from 'react-dom/client';
import { QueryClient, QueryClientProvider } from '@tanstack/react-query';
import { RouterProvider, createRouter } from '@tanstack/react-router';
import { ConfigProvider, App as AntApp, theme } from 'antd';
import { I18nextProvider } from 'react-i18next';
import {
  AbilityProvider,
  createIcoreApi,
  createIcoreI18n,
  ICORE_LOCALES,
} from '@icore/template-shared';
import { routeTree } from './routeTree.gen';

const queryClient = new QueryClient({
  defaultOptions: { queries: { retry: false, refetchOnWindowFocus: false } },
});

const router = createRouter({ routeTree, context: { queryClient } });
declare module '@tanstack/react-router' {
  interface Register {
    router: typeof router;
  }
}

const i18n = createIcoreI18n({ resources: ICORE_LOCALES });

export const api = createIcoreApi({
  baseUrl: import.meta.env.VITE_API_URL ?? '/api',
  onUnauthorized: () => router.navigate({ to: '/login' }),
});

createRoot(document.getElementById('root')!).render(
  <StrictMode>
    <I18nextProvider i18n={i18n}>
      <ConfigProvider theme={{ algorithm: theme.defaultAlgorithm }}>
        <AntApp>
          <QueryClientProvider client={queryClient}>
            <AbilityProvider>
              <RouterProvider router={router} />
            </AbilityProvider>
          </QueryClientProvider>
        </AntApp>
      </ConfigProvider>
    </I18nextProvider>
  </StrictMode>,
);
```

`<AntApp>` from antd 6 enables the `useApp()` hook → message/notification/modal pulled from React tree. We use it inside MainLayout to call `notification.useNotification()` and bind.

## Task 6: PageLayout + AccessDeniedPage

`apps/templates/client-antd/src/components/PageLayout.tsx`:

```tsx
import type { ReactNode } from 'react';
import { Descriptions, Spin, Space } from 'antd';
import { Can, useDraft, useLoading } from '@icore/template-shared';
import { AccessDeniedPage } from './AccessDeniedPage';
import type { AbilityAction, AbilitySubject } from '@icore/shared';

export interface PageLayoutProps {
  title: ReactNode;
  description?: ReactNode;
  action?: AbilityAction;
  subject?: AbilitySubject;
  extra?: ReactNode;
  children?: ReactNode;
}

export function PageLayout({
  title,
  description,
  action = 'read',
  subject = 'all',
  extra,
  children,
}: PageLayoutProps) {
  useDraft(false);
  const loading = useLoading();

  return (
    <Can I={action} a={subject} passThrough>
      {(allowed) =>
        allowed ? (
          <div style={{ padding: 24 }}>
            <Descriptions title={title} extra={extra}>
              {description ? <Descriptions.Item>{description}</Descriptions.Item> : null}
            </Descriptions>
            <Spin spinning={loading}>
              <div>{children}</div>
            </Spin>
          </div>
        ) : (
          <AccessDeniedPage />
        )
      }
    </Can>
  );
}
```

`AccessDeniedPage.tsx`:

```tsx
import { Result, Button } from 'antd';
import { Link } from '@tanstack/react-router';

}
```
