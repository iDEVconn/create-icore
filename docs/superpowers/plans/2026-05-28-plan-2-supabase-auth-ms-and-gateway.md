# Plan 2: Supabase Auth MS + Gateway with AuthGuard + CASL

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stand up the NestJS gateway (`apps/api`) and the auth microservice (`apps/microservices/auth`), wire the Supabase auth strategy (`libs/auth-strategies/supabase`) on top of `@idevconn/supabase`, connect them via the configurable transport from `@icore/shared`, and protect the gateway with `AuthGuard` + CASL `AbilityGuard`. After this plan, `POST /api/auth/register → /login → GET /api/profile` works end-to-end against a real Supabase project, and the `SupabaseAuthStrategy` passes the existing `runAuthContract` suite.

**Architecture:** Gateway exposes HTTP, marshals requests over TCP (default) into the auth MS via the new `libs/auth-client`. The MS hosts `@MessagePattern` handlers backed by an `AuthStrategy` factory provider. The Supabase concrete strategy adapts `@idevconn/supabase`'s `AuthService` to the icore `AuthStrategy` shape. CASL rules are loaded from `@icore/shared`'s `defineAbilitiesFor` on every authenticated request.

**Tech Stack:** NestJS 11, `@nestjs/microservices`, `@nestjs/throttler`, `@nestjs/config`, `@casl/ability`, `@idevconn/supabase` (^0.12), `@supabase/supabase-js` (transitive via `@idevconn/supabase`).

**Source spec:** `docs/superpowers/specs/2026-05-28-icore-design.md`

**Branch:** `dev`. Plan 1 HEAD: `7b4691d`.

**Generators only** (per `AGENTS.md`). Hand-crafted `project.json` is forbidden.

```ts
import type { SupabaseClient } from '@supabase/supabase-js';

interface FakeUser {
  id: string;
  email: string;
  password: string;
  role?: string;
}

interface FakeSession {
  access_token: string;
  refresh_token: string;
  expires_in: number;
  user: { id: string; email: string };
}

export function createMockSupabaseClient(): SupabaseClient {
  const users = new Map<string, FakeUser>();
  const accessToUid = new Map<string, string>();
  const refreshToUid = new Map<string, string>();

  function issueSession(user: FakeUser): FakeSession {
    const access_token = `at_${user.id}_${Date.now()}_${Math.random()}`;
    const refresh_token = `rt_${user.id}_${Date.now()}_${Math.random()}`;
    accessToUid.set(access_token, user.id);
    refreshToUid.set(refresh_token, user.id);
    return {
      access_token,
      refresh_token,
      expires_in: 3600,
      user: { id: user.id, email: user.email },
    };
  }

  function findById(uid: string): FakeUser | undefined {
    return [...users.values()].find((u) => u.id === uid);
  }

  const auth = {
    async signUp({ email, password }: { email: string; password: string }) {
      if ([...users.values()].some((u) => u.email === email)) {
        return { data: { user: null, session: null }, error: { message: 'user exists' } };
      }
      const user: FakeUser = { id: `uid_${users.size + 1}`, email, password };
      users.set(user.id, user);
      const session = issueSession(user);
      return { data: { user: session.user, session }, error: null };
    },
    async signInWithPassword({ email, password }: { email: string; password: string }) {
      const user = [...users.values()].find((u) => u.email === email);
      if (!user || user.password !== password) {
        return { data: { user: null, session: null }, error: { message: 'invalid credentials' } };
      }
      const session = issueSession(user);
      return { data: { user: session.user, session }, error: null };
    },
    async refreshSession({ refresh_token }: { refresh_token: string }) {
      const uid = refreshToUid.get(refresh_token);
      if (!uid) return { data: { session: null }, error: { message: 'invalid refresh' } };
      refreshToUid.delete(refresh_token);
      const user = findById(uid);
      if (!user) return { data: { session: null }, error: { message: 'user missing' } };
      const session = issueSession(user);
      return { data: { session }, error: null };
    },
    async getUser(token: string) {
      const uid = accessToUid.get(token);
      if (!uid) return { data: { user: null }, error: { message: 'invalid token' } };
      const user = findById(uid)!;
      return {
        data: {
          user: { id: user.id, email: user.email, app_metadata: { role: user.role } },
        },
        error: null,
      };
    },
    admin: {
      async updateUserById(uid: string, updates: { app_metadata?: { role?: string } }) {
        const user = findById(uid);
        if (!user) return { data: { user: null }, error: { message: 'user missing' } };
        if (updates.app_metadata?.role) user.role = updates.app_metadata.role;
        return { data: { user: { id: user.id, email: user.email } }, error: null };
      },
    },
  };

  return { auth } as unknown as SupabaseClient;
}
```

```ts
import type { SupabaseClient } from '@supabase/supabase-js';
import type { AuthSession, AuthStrategy, VerifiedToken } from '@icore/shared';

export interface SupabaseAuthStrategyOptions {
  client: SupabaseClient;
}

export class SupabaseAuthStrategy implements AuthStrategy {
  private readonly client: SupabaseClient;

  constructor(opts: SupabaseAuthStrategyOptions) {
    this.client = opts.client;
  }

  async signUp(email: string, password: string): Promise<AuthSession> {
    const { data, error } = await this.client.auth.signUp({ email, password });
    if (error || !data.session) throw new Error(error?.message ?? 'signup_failed');
    return this.toSession(data.session);
  }

  async signIn(email: string, password: string): Promise<AuthSession> {
    const { data, error } = await this.client.auth.signInWithPassword({ email, password });
    if (error || !data.session) throw new Error(error?.message ?? 'invalid_credentials');
    return this.toSession(data.session);
  }

  async refresh(refreshToken: string): Promise<AuthSession> {
    const { data, error } = await this.client.auth.refreshSession({ refresh_token: refreshToken });
    if (error || !data.session) throw new Error(error?.message ?? 'invalid_refresh_token');
    return this.toSession(data.session);
  }

  async verifyToken(token: string): Promise<VerifiedToken> {
    const { data, error } = await this.client.auth.getUser(token);
    if (error || !data.user) throw new Error(error?.message ?? 'invalid_token');
    return {
      uid: data.user.id,
      email: data.user.email,
      role: (data.user.app_metadata as { role?: string } | undefined)?.role,
    };
  }

  async setRole(uid: string, role: string): Promise<void> {
    const { error } = await this.client.auth.admin.updateUserById(uid, {
      app_metadata: { role },
    });
    if (error) throw new Error(error.message);
  }

  private toSession(s: {
    access_token: string;
    refresh_token: string;
    expires_in: number;
    user: { id: string; email?: string };
  }): AuthSession {
    return {
      accessToken: s.access_token,
      refreshToken: s.refresh_token,
      expiresIn: s.expires_in,
      user: { id: s.user.id, email: s.user.email ?? '' },
    };
  }
}
```

Create `apps/microservices/auth/src/app/auth.controller.ts`:

```ts
import { Controller, Inject } from '@nestjs/common';
import { MessagePattern, Payload } from '@nestjs/microservices';
import type { AuthSession, AuthStrategy, VerifiedToken } from '@icore/shared';

@Controller()
export class AuthController {
  constructor(@Inject('AuthStrategy') private readonly strategy: AuthStrategy) {}

  @MessagePattern('auth.verify')
  verify(@Payload() { token }: { token: string }): Promise<VerifiedToken> {
    return this.strategy.verifyToken(token);
  }

  @MessagePattern('auth.login')
  login(@Payload() { email, password }: { email: string; password: string }): Promise<AuthSession> {
    return this.strategy.signIn(email, password);
  }

  @MessagePattern('auth.signup')
  signup(@Payload() { email, password }: { email: string; password: string }): Promise<AuthSession> {
    return this.strategy.signUp(email, password);
  }

  @MessagePattern('auth.refresh')
  refresh(@Payload() { refreshToken }: { refreshToken: string }): Promise<AuthSession> {
    return this.strategy.refresh(refreshToken);
  }

  @MessagePattern('auth.setRole')
  setRole(@Payload() { uid, role }: { uid: string; role: string }): Promise<void> {
    return this.strategy.setRole(uid, role);
  }
}
```

```ts
import {
  CanActivate,
  ExecutionContext,
  Inject,
  Injectable,
  UnauthorizedException,
} from '@nestjs/common';
import { Reflector } from '@nestjs/core';
import { AuthClientService } from '@icore/auth-client';
import type { Request } from 'express';
import { IS_PUBLIC_KEY } from './public.decorator';

@Injectable()
export class AuthGuard implements CanActivate {
  constructor(
    private readonly reflector: Reflector,
    @Inject(AuthClientService) private readonly authClient: AuthClientService,
  ) {}

  async canActivate(ctx: ExecutionContext): Promise<boolean> {
    const isPublic = this.reflector.getAllAndOverride<boolean>(IS_PUBLIC_KEY, [
      ctx.getHandler(),
      ctx.getClass(),
    ]);
    if (isPublic) return true;

    const req = ctx.switchToHttp().getRequest<Request & { user?: unknown }>();
    const header = req.headers.authorization ?? '';
    const [scheme, token] = header.split(' ');
    if (scheme !== 'Bearer' || !token) throw new UnauthorizedException('missing_bearer');

    try {
      const verified = await this.authClient.verify(token);
      req.user = verified;
      return true;
    } catch {
      throw new UnauthorizedException('invalid_token');
    }
  }
}
```

```ts
import { CanActivate, ExecutionContext, ForbiddenException, Injectable } from '@nestjs/common';
import { Reflector } from '@nestjs/core';
import type { Request } from 'express';
import type { VerifiedToken } from '@icore/shared';
import { AbilityFactory } from './ability.factory';
import { CHECK_ABILITY_KEY, type RequiredRule } from './check-ability.decorator';

@Injectable()
export class AbilityGuard implements CanActivate {
  constructor(
    private readonly reflector: Reflector,
    private readonly factory: AbilityFactory,
  ) {}

  canActivate(ctx: ExecutionContext): boolean {
    const required = this.reflector.getAllAndOverride<RequiredRule | undefined>(
      CHECK_ABILITY_KEY,
      [ctx.getHandler(), ctx.getClass()],
    );
    if (!required) return true;

    const req = ctx.switchToHttp().getRequest<Request & { user?: VerifiedToken }>();
    const ability = this.factory.forUser(req.user ?? null);
    if (!ability.can(required.action, required.subject)) {
      throw new ForbiddenException();
    }
    return true;
  }
}
```

## Self-Review Notes

**Spec coverage (Plan 1 design spec):**

- Gateway with `AuthGuard`, `@Public`, ThrottlerModule, CASL `AbilityGuard`, `@CheckAbility` decorator → Tasks 7–8 ✅
- Auth MS with `@MessagePattern` handlers (`auth.verify`, `auth.login`, `auth.signup`, `auth.refresh`, `auth.setRole`) → Task 5 ✅
- `libs/auth-strategies/supabase` adapting `@idevconn/supabase` → Tasks 3–4 ✅
- `libs/auth-client` exposing `forRoot()` + typed service → Task 6 ✅
- Transport via `buildTransport('AUTH')` from `@icore/shared` → Tasks 2, 6 ✅
- 3-layer env (transport / provider selection / provider credentials) → Tasks 5, 7, 9 ✅
- Per-MS `.env` loaded via `ConfigModule.forRoot({envFilePath})` → Task 5 Step 2 ✅
- `runAuthContract` runs against the real Supabase strategy → Task 4 ✅

**Deliberately deferred:**

- Firebase auth strategy → Plan 3.
- Storage MS → Plan 4.
- Client app + `<AbilityProvider>` + `<Can>` → Plan 6.
- Docker-compose for local dev → Plan 7 or later.
- CI matrix (`AUTH_PROVIDER=[supabase,firebase]`) → Plan 3.
- Real Supabase live test (without mocked client) → optional E2E job once a Supabase test project is provisioned.

**Type consistency:**

- `AuthStrategy`, `AuthSession`, `VerifiedToken`, `runAuthContract`, `FakeAuthStrategy` consistent across all tasks.
- `AuthClientService.verify(token)` returns `VerifiedToken`; gateway `AuthGuard` consumes the same type and attaches it as `req.user`; `AbilityFactory.forUser(token)` accepts it. Single shape end-to-end.
- `@MessagePattern` names: `auth.verify` / `auth.login` / `auth.signup` / `auth.refresh` / `auth.setRole`. Used identically in MS controller, client service, and contract tests.

**Out of scope (already excluded from icore v0.1.0 by spec Non-Goals):**

- OAuth providers (Google, GitHub) — `@idevconn/supabase` supports OAuth in its `AuthService`, but icore's `AuthStrategy` is password-only for v0.1.0. OAuth lands when a consumer needs it.
- Magic link / OTP login — same reasoning.
- Subscriptions / entitlements layer — explicit spec non-goal.
