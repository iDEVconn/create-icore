# Plan 6.3: Theme Switching (light / dark) Across All Templates

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a unified light/dark theme toggle that works identically across all three UI templates (shadcn / antd / mui). The choice persists across reloads via localStorage and respects the OS preference on first load. Each template wires its library-specific theming primitives to a shared store; app code reads + toggles via a single `useTheme()` hook from `@icore/template-shared`.

**Architecture:**

- **Shared store** — `libs/template-shared/src/lib/stores/theme.store.ts` exports a Zustand store with `mode: 'light' | 'dark'`, `setMode(m)`, and `toggle()`. Persisted to localStorage key `icore-theme`. First load defaults to `prefers-color-scheme: dark` media query, else `'light'`.
- **`useTheme()` hook** — convenience re-export from `@icore/template-shared`.
- **Per-template wiring** — each template reads `useTheme()`:
  - **shadcn** — toggles `class="dark"` on `document.documentElement`. Tailwind 4 `@theme` blocks already define light + dark token sets; toggle is a single class.
  - **antd** — passes `theme.darkAlgorithm` to `ConfigProvider.theme.algorithm` when dark.
  - **mui** — passes `mode: 'dark'` to `createTheme.palette` when dark.
- **`<ThemeToggle />` component** — each template ships its own (library-specific button + icon) that calls `useTheme().toggle()`. All three live in the MainLayout's LayoutHeader.

**Tech Stack:** No new deps. Reuses the existing template-shared Zustand setup + each template's existing theme primitives.

## Task 1: Theme store in `@icore/template-shared`

Create `libs/template-shared/src/lib/stores/theme.store.ts`:

```ts
import { create } from 'zustand';
import { persist } from 'zustand/middleware';

export type ThemeMode = 'light' | 'dark';

interface ThemeState {
  mode: ThemeMode;
  setMode: (m: ThemeMode) => void;
  toggle: () => void;
}

function detectInitial(): ThemeMode {
  if (typeof window === 'undefined') return 'light';
  return window.matchMedia?.('(prefers-color-scheme: dark)').matches ? 'dark' : 'light';
}

export const useThemeStore = create<ThemeState>()(
  persist(
    (set, get) => ({
      mode: detectInitial(),
      setMode: (mode) => set({ mode }),
      toggle: () => set({ mode: get().mode === 'dark' ? 'light' : 'dark' }),
    }),
    { name: 'icore-theme' },
  ),
);

export const useTheme = () => useThemeStore();
```

Add to `libs/template-shared/src/index.ts`:

```ts
export * from './lib/stores/theme.store';
```

// Apply the theme class before React mounts so the first paint is correct
const applyTheme = (mode: 'light' | 'dark') => {
  document.documentElement.classList.toggle('dark', mode === 'dark');
};
applyTheme(useThemeStore.getState().mode);
useThemeStore.subscribe((s) => applyTheme(s.mode));

import { ConfigProvider, App as AntApp, theme } from 'antd';
import { useThemeStore } from '@icore/template-shared';

function Root() {
  const mode = useThemeStore((s) => s.mode);
  const algorithm = mode === 'dark' ? theme.darkAlgorithm : theme.defaultAlgorithm;

## Task 4: MUI theme wiring

### Step 1: ThemeProvider with mode from store

Edit `apps/templates/client-mui/src/main.tsx` — make `createTheme` reactive to the store. Same Root-component pattern as antd:

```tsx
import { CssBaseline, ThemeProvider, createTheme } from '@mui/material';
import { useThemeStore } from '@icore/template-shared';
import { useMemo } from 'react';

function Root() {
  const mode = useThemeStore((s) => s.mode);
  const theme = useMemo(
    () => createTheme({ palette: { mode, primary: { main: '#5e60ce' } } }),
    [mode],
  );
  return (
    <ThemeProvider theme={theme}>
      <CssBaseline />
      <QueryClientProvider client={queryClient}>
        <AbilityProvider>
          <RouterProvider router={router} />
        </AbilityProvider>
      </QueryClientProvider>
    </ThemeProvider>
  );
}

createRoot(document.getElementById('root')!).render(
  <StrictMode>
    <I18nextProvider i18n={i18n}>
      <Root />
    </I18nextProvider>
  </StrictMode>,
);
```
